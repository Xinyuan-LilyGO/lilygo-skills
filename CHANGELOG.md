# Changelog

## Unreleased

- Rewrote the runtime from the Rust CLI to a JS thin core: a Node dispatcher
  under `bin/*.mjs` with no build step, types provided by JSDoc plus `tsc`.
- Kept the AI-facing command surface: `context`, `source query`,
  `verify sources`, `doctor`, and `hook <claude|codex>` behave as before through
  the `lilygo-skills` shim.
- Retired the maintainer commands that shipped with the Rust CLI (`route`,
  `goal`, `index`, `benchmark`, `sync-boards`, `source completeness`, and the
  `update *` family). Board and fact-pack data are now regenerated by the
  official-source pipeline (`node pipeline/run-official-source-pipeline.js`).
- Made the data model travel with the skill directory: the installer mounts
  `bin/**` and `data/**` together, so the runtime is self-contained.
- Moved the ci-gate suite to a JS baseline; verification runs `npm test`,
  `npx tsc --noEmit`, and `bash scripts/ci-gate.sh`.
- Held effect parity on installed-state runs: board-triple values 12/12 not
  worse, verifiable-strict 7/12 versus the bare-model 3/12, and 0 hallucinated
  pins/buses.
- Added the official-source pipeline, byte-for-byte capsule gates, board
  triple-question evaluation, and private scorecard fail-closed checks.
- Added source-backed T-Watch S3 display and touch facts, Chinese screen/touch
  routing, and narrow display/input source-query expansion.
- Aligned README and architecture docs around the same L0-L14 progressive
  disclosure model.
- Preserved the verification boundary: these updates are source/context,
  install, generation, and evaluation evidence unless a separate V4/V5 hardware
  artifact is attached.

## 0.1.0 - 2026-07-02

- Added the LilyGO Skill runtime with a Rust CLI, meta Skill entry, generated
  runtime skills, source facts, setup planning, goal planning, and benchmark
  gates.
- Kept the public source tree meta-only: generated board, framework,
  peripheral, chip, feature, playbook, and app skills are runtime artifacts
  produced from `data/**`, `skills/references/**`, and `templates/skills/**`.
- Limited runnable guidance to LilyGO products in the ESP32 family until other
  families have their own design and verification evidence.
- Added release-hardening checks for mixed ASCII/CJK prompts, T-Display-S3
  display source completeness, and build-capable installation.
