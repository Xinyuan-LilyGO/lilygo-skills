# Skills Directory — meta-only source tree

This directory is **meta-only**. The only committed Skill is the meta router
`lilygo-router/SKILL.md`, which tells an agent how to ask the `lilygo-skills`
CLI for compact context, source-cited facts, and verification actions.
Static reference docs under `references/` are support material, not routed
Skills.

Generated board, series, framework, tool, peripheral, chip, feature, debug,
app/recipe, and playbook skills are **not committed here**. They are runtime
artifacts produced from the source model (`data/**`). The board capsule is
surfaced on demand:

```bash
# Resolve the board and thin capsule for a prompt or project:
lilygo-skills context --json "<prompt>"

# Pull exact source-cited facts, and re-prove them against their sources:
lilygo-skills source query --board <board-id> --topic <topic> --json
lilygo-skills verify sources --board <board-id> --json
```

The installer (`node install.js`) mounts the runtime (`bin/**` + `data/**`) into
the selected Codex/Claude install root. `context` and `hook` calls stay no-write:
they may report a follow-up command, but never write files implicitly. Board and
fact-pack data are regenerated by the official-source pipeline
(`node pipeline/run-official-source-pipeline.js`).

Source-backed facts live under `data/`, board triggers live in
`index/routes.json`, reference practice skills live under
`data/skills/reference/`, playbooks live under `data/playbooks/`, and public
references live in official repos, docs, examples, datasheets, or project-local
`.lilygo-skills/references.json`.

Generated Skill file shapes live under `../templates/skills/`. The pipeline uses
those templates for board, peripheral/chip/feature, and playbook output, and the
installer copies both templates and `references/` into the install roots.

Why meta-only: committing generated snapshots turns the repository into an opaque
dump of hand-written skill text. Keeping only the source model plus a
deterministic pipeline keeps the release auditable and the skills reproducible.
